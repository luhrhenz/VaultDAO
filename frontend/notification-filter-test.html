🧪 Notification Filter Sync Test Suite

Test 1: Filter Sync on Panel Open

Objective: Verify that opening the panel shows current filter state
  1. Start your development server: npm run dev
  2. Open the application in your browser
  3. Open the NotificationCenter panel (click the bell icon)
  4. Click "Filter" to open the filter panel
  5. Verify all default filters are selected
Expected: All categories (Proposals, Approvals, System) and all priorities should be selected by default

Test 2: External Filter Reset Sync

Objective: Verify external filter resets are reflected when panel reopens
  1. Open NotificationCenter and deselect "Proposals" category
  2. Click "Apply Filters"
  3. Close the NotificationCenter panel
  4. Click the "Reset" button in the filter panel (or trigger reset externally)
  5. Reopen the NotificationCenter panel
  6. Open the filter panel again
  7. Verify "Proposals" is now selected again
Expected: The filter should reflect the reset state, with "Proposals" selected again

Test 3: No Sync When Panel Already Open

Objective: Verify selections aren't interrupted while panel is open
  1. Open NotificationCenter panel
  2. Open filter panel
  3. Deselect "Proposals" (don't apply yet)
  4. Keep panel open and interact with other UI elements
  5. Return to filter panel
  6. Verify "Proposals" is still deselected
Expected: User's selection should be preserved (no unwanted sync)
